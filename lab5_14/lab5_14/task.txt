Тема работы: Обмен информацией с дисковыми файлами,

Цель работы: получить практические знания и овладеть средствами высоко-уровневого 
буферизированная обмена данными с файлами; научиться создает ваты, обрабатывать, 
переименовывать и удалять дисковые файлы, испольпользуя библиотечные функции языка С.
Методические указания 1. Внимательно прочитать условие задачи. Если в задаче требуется 
обработать файл, который уже иснуюе, то необходимо предварительно создать соответствующий 
файл. Для создания текстовых файлов можно воспользоваться средствами одного из текстовых 
редакторов. Бинарные файлы надо создавать программно.
2. Созданный файл (файлы) лучше записать в свой рабочий каталог, из которого будет 
запускаться программа, и сделать его активным каталогом. В противном случае необходимо 
указывать полный или сокращенный путь к файлу.
3. Если в условии задачи не указано имя файла или способ формирования имени, то 
целесообразно предоставить пользователю возможность задавать имя файла (или имена файлов) 
в списке параметров комадно строки, считывая эти имена через параметры функции main (). 
В программе надо обязательно выполнить проверку: если в комадно строке имя файла 
не указано, то обеспечить возможность ввода имени файла с клавиатуры.
4. Большинство задач требуют редактирования содержимого файла. Для этого следует 
реализовать следующие действия
1) открыть файл, который должен быть проработан для чтения;
2) создать новый файл для записи;
3) последовательно считывать информацию с базового файла, видоизменять ее согласно
условию задачи и записывать в новый файл;
4) закрыть оба файла;
5) если результат изменения должен быть сохранен в файле с исходным именем, то базовый 
файл надо вытереть, а вновь - переименовать;
6) можно не вытирать базовый файл, а только изменить его имя или расширение.
5. Для обработки содержимого файла необходимо разработать соответствующие функции.
6. Считывание информации из файла и запись данных в файл реализовать с помощью библиотечных 
функций высокоуровневого буферизированная обмена данными.
7. Если по условию задачи нужно создать бинарный файл, то в такой файл надо записывать 
внутренние коды соответствующих данных (переписать их из оперативной памяти). Лучше всего 
для этого воспользоваться стандартной функцией fwrite (), а для считывания данных - 
функцией fread (). В случае записи в бинарный файл, считывание файла данных, имеющих 
тип int, можно использовать также функции putw () и getw ().
8. Работа программы должно сопровождаться выводом на экран соответствующих сообщений, 
в том числе о введении имени файла или входных данных, создания нового файла или изменения 
имени базового файла и тому подобное.
9. Если программа редактирует файл создает новый текстовый файл, то нужно пересмотреть его 
и проверить правильность заполнения. Если же результатом работы программы является бинарный 
файл, то для контроля правильности надо обязательно распечатать содержимое файла в удобной 
для чтения форме.

14. Разработать функцию, которая осуществляет замену в заданном предложении 
всех малых украинских и латинских букв в соответствующие заглавные буквы. 
Подсказка: в функции воспользоваться массивами больших и малых букв украинского 
алфавита. Используя разработанную функцию, переписать заглавными буквами в 
новый файл пять первых предложений заданного текстового файла (конец предложения - 
точка с последующим пробелом, табуляцией или символом новой строки).
